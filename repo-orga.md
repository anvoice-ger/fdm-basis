# Dateibenennung

Die **template-Dateien** sind nach folgendem Muster benannt:

- **template_I.md**
- **template_A.md**
- usw.

Dabei gilt:

- A = Aktivität/Aufgabe
- I = Input
- usw.

Die **bricks** sind folgendermaßen benannt:

Beispiel: [lzm-id]\_[Stichwort-Inhalt]\_[Vermittlungsform]\_[Sozialform].md

Dabei gilt:

- Vermittlungsform: [A oder I]

  - A = Aktivität/Aufgabe
  - I = Input

Die **builds** (Zusammenstellung) der bricks gilt
  
überlegmalwas.md
